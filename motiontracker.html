Purpose

Motion Tracker Lab is a browser-based physics video analysis tool for manually tracking the motion of an object in a video. It helps users convert video observations into calibrated position, velocity, acceleration, tables, graphs, and curve fits.

Target Users

  • Physics students collecting position-time data from real videos.
  • Physics teachers and laboratory instructors running mechanics activities.
  • STEM learners working on classroom, home, or remote experiments.
  • Schools that need a no-install browser tool for shared devices.

Common Use Cases

  • Projectile motion analysis from tossed balls or launched objects.
  • Cart acceleration on ramps and other constant-acceleration experiments.
  • Pendulum, oscillation, and simple collision investigations.
  • Exporting experimental motion data for reports or spreadsheets.

Core Workflow

  1. Upload a video file or load the built-in sample video.
  2. Set scale using two video points with a known real-world distance.
  3. Set the coordinate origin.
  4. Set the positive x-axis direction and coordinate convention.
  5. Track the object's position frame by frame.
  6. Review the data table, graphs, and optional curve fits.
  7. Export data as CSV, copy table data, or export a graph image.

Main Features

  • Local video loading for MP4, WebM, and browser-supported MOV files.
  • Frame stepping, playback controls, scrubber, and configurable FPS.
  • Scale, origin, and axis calibration with physics or screen coordinates.
  • Manual point tracking with optional auto-advance.
  • Overlay markers for calibration, axes, tracked points, and trajectory.
  • Zoom and pan controls for precise point selection.
  • Data table with time, pixel coordinates, real coordinates, and velocities.
  • Graphs for position, trajectory, velocity, and acceleration.
  • Linear and quadratic fits with R-squared and physics interpretations.
  • CSV export, graph PNG export, and clipboard copy.

Limitations

  • Tracking is manual; automatic object detection is not included.
  • The app is designed for one tracked object at a time.
  • Accuracy depends on correct FPS selection, video quality, and click precision.
  • Large videos and some MOV codecs may not load smoothly in every browser.
  • Measurement uncertainty and graph error bars are not currently included.
